# ADR-003: Test-Driven Development

**Status:** Accepted

**Decision:** All core functionality is written test-first. No production code in `src/core/` without a preceding failing test. Shell code is not unit-tested.

## Workflow

1. Write a failing test describing the desired behavior **(red)**
2. Write the minimal core code to pass it **(green)**
3. Refactor — names, structure, duplication — tests must stay green
4. Commit

Skipping red is a standards violation. "I'll add the test later" is not permitted.

## Test Scope

| Code | Tested? | How |
|---|---|---|
| `src/core/` | Yes — all exported functions | Vitest unit tests, no mocks |
| `src/shell/` | No unit tests | Manual or future E2E |

Tests live adjacent to the code they test:
- `src/core/session/session.ts` → `src/core/session/session.test.ts`

## Coverage Policy

- Every exported core function has tests
- Happy path covered
- All predictable failure and edge states covered (including Null Object returns)
- No numeric coverage target — aim for "no untested behavior that could fail silently"
- Running tests is part of the development loop, not a gate at the end

## Test Anatomy

```typescript
describe('computeWeightDelta', () => {
  it('returns grams difference between two weight entries', () => {
    const result = computeWeightDelta(entryAt(100), entryAt(85))
    expect(result.grams).toBe(15)
  })

  it('returns NullWeightDelta when the earlier entry is absent', () => {
    const result = computeWeightDelta(NullWeightEntry, entryAt(85))
    expect(result).toBe(NullWeightDelta)
  })
})
```

## Anti-patterns

- Tests written after the fact ("retrofitted") — these are not TDD
- Tests that only cover the happy path
- Mocking core functions to test other core functions — core is pure, no mocks needed
- Tests in `src/shell/`
- Tests that assert implementation details rather than observable behavior
- Test descriptions that say "it calls X" rather than "it returns Y when Z"
